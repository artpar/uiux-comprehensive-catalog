# Product

## Register

product

## Users

UI builders and reviewers use this catalog when they need to choose, justify, implement, or critique an interface pattern. The primary users are frontend engineers, product designers, product teams, and AI agents that consume the generated exports before producing UI code.

## Product Purpose

The catalog provides source-backed guidance for practical UI and UX patterns. It helps users start from the interaction problem, compare plausible alternatives, inspect anti-pattern risks, and carry implementation-ready decision language into code review or agent prompts. Success means the site feels trustworthy, readable, and usable enough that teams can rely on it before building.

## Brand Personality

Credible, practical, approachable. The voice should be direct and expert without becoming academic, dense, or intimidating.

## Anti-references

Avoid cluttered, claustrophobic, scary, content-vomit pages. Avoid generic AI/SaaS template aesthetics, decorative trend surfaces, and interfaces that claim UX leadership while making the catalog difficult to scan, compare, or trust.

## Design Principles

- Practice what the catalog teaches: every page should demonstrate clear hierarchy, progressive disclosure, and task-fit pattern selection.
- Make evidence usable: source-backed content should be easy to scan before it is exhaustive.
- Keep the lab and catalog distinct: readable browsing and interactive decision work should support each other without one swallowing the other.
- Prefer expert calm over spectacle: the interface should feel confident, restrained, and useful under repeated use.
- Design for agents and humans: guidance should remain precise enough for implementation prompts while staying approachable for product teams.

## Accessibility & Inclusion

Aim for WCAG 2.2 AA quality. Preserve keyboard access, visible focus, readable contrast, responsive layouts, reduced-motion safety, clear labels, and source-backed interaction semantics. The product should be especially careful with pages that teach accessibility-sensitive patterns such as dialogs, navigation, selection controls, validation, status messages, and AI transparency.
